# 주식 매매 일지 (Stock Trading Journal) - Multi-User Edition

[![License](https://img.shields.io/badge/License-Apache_2.0-blue.svg)](https://opensource.org/licenses/Apache-2.0)

개인 투자자를 위한 **로컬 기반 주식 매매 일지 및 포트폴리오 관리 웹 애플리케이션**입니다.
파이썬(Flask) 백엔드와 SQLite를 기반으로 하며, **다중 사용자(Multi-User) 환경**과 **모바일 기기 최적화**가 적용되어 언제 어디서든 안전하고 쾌적하게 매매 기록을 관리할 수 있습니다.

## 1. 프로그램 개요 및 목적 (Overview & Objective)

이 프로그램은 개인 투자자가 엑셀이나 수기 노트 대신 **웹 브라우저를 통해 직관적이고 체계적으로 매매 일지를 작성하고 포트폴리오를 관리**할 수 있도록 돕는 도구입니다.
단순한 기록을 넘어, **매매 성과 분석(승률, 손익비 등), 캘린더 뷰, 실시간 현재가 및 뉴스 연동** 기능을 제공하여 투자 복기와 전략 수립을 극대화하는 데 목적이 있습니다.

## 2. 주요 기능 상세 (Key Features)

*   **다중 사용자 & 보안 (Multi-User & Security)**
    *   회원가입 및 로그인 기능을 제공하며, 최초 가입자가 자동으로 최고 관리자로 지정되어 이후 가입자에 대한 승인 제어가 가능합니다.
    *   로그인 5회 실패 시 1분간 계정 차단, 1시간 장기간 미활동 시 5분 전 경고 및 자동 로그아웃 처리.
    *   대용량 파일 업로드 방어(16MB 제한) 및 XSS, CSRF 보안 조치 등 안전한 금융 데이터 보호가 적용되어 있습니다.
*   **매매 일지 & 스마트 에디터 (Journal & Smart Editor)**
    *   매수, 매도, 관망, 배당 기록뿐 아니라 종목에 대한 일반 메모와 아이디어를 기록할 수 있습니다.
    *   에디터 본문 내에 이미지 직접 삽입(스크린샷 클립보드 붙여넣기, 드래그 앤 드롭 지원) 및 크기 조절이 가능하며, 외부 텍스트 붙여넣기 시 색상/배경 서식을 자동 정제합니다.
*   **포트폴리오 대시보드 & 현재가 연동 (Dashboard & Real-time Price)**
    *   현재 보유 종목, 총 투자 금액, 평균 단가, 누적 실현 손익을 자동 계산하고 시각적인 파이 차트로 제공합니다. (사용자 지정 드래그 앤 드롭 정렬 지원)
    *   1분 주기 현재가 자동 업데이트를 지원하며, 정규장(KRX) 뿐만 아니라 **시간외 단일가(NXT)** 캐싱 및 토글 기능을 완벽하게 지원합니다.
    *   청산 종목 보기, 현재가 보기 등의 옵션 상태가 사용자별 설정(Preferences)에 동기화됩니다.
*   **5중 독립 필터링 (Advanced Filtering)**
    *   기록별, 종목별, 계좌별, 증권사별, 투자 분류별 5개의 독립된 필터를 통해 방대한 매매 기록을 신속하고 정밀하게 필터링하여 분석할 수 있습니다.
*   **매매 성과 분석 & 차트 뷰 (Performance Analytics)**
    *   승률, 손익비(Profit Factor), 평균 수익/손실, 최대 낙폭(MDD), 평균 보유기간 등 회고를 위한 심도 있는 통계 지표를 제공합니다.
    *   월간/주간 단위 실현 손익, 평가 손익, 매매 금액, 누적 수익금 흐름을 보여주는 인터랙티브 막대 차트를 지원하며, 차트 막대 클릭 시 해당하는 세부 거래 내역을 바로 확인할 수 있습니다.
*   **데이터 무결성 검증 (Data Integrity)**
    *   보유 수량을 초과하는 매도나 미보유 종목에 대한 매도 등, 논리적으로 잘못된 데이터 입력을 서버 단계에서 원천 차단하여 평균 단가와 실현 손익의 무결성을 유지합니다.
*   **캘린더 뷰 & 엑셀 내보내기 (Calendar & Export)**
    *   월별 달력에서 일자별 매매/메모 현황과 실현 손익을 색상 테마별로 직관적으로 파악할 수 있으며, 전체 기록을 버튼 한 번에 엑셀(XLSX) 데이터로 내보낼 수 있습니다.
*   **실시간 뉴스 연동 (Live News Integration)**
    *   보유 중이거나 최근 기록한 종목과 관련된 최신 뉴스를 구글 뉴스(RSS) 기반으로 5분 주기로 로드하여 사이드바에 제공합니다.
*   **관리자 대시보드 & 시스템 로깅 (Admin & Logging)**
    *   전체 사용자 목록, 가입/최근 접속 시간, 데이터 통계를 한눈에 확인하고 사용자 상태를 관리할 수 있는 관리자 전용 대시보드를 지원합니다.
    *   접속, 작업, 에러 로그가 매일 날짜별 파일(`logs/backend_app_*.log`)로 자동 순환 기록되며 유지보수가 간편합니다.
*   **모바일 최적화 (PWA Ready)**
    *   데스크톱/모바일 UI 자동 반응형 대응, 다크/라이트 테마 지원, iOS/Android 기기에서 홈 화면 추가 시 네이티브 앱처럼 동작하는 쾌적한 UX를 제공합니다.

## 4. 설치 및 실행 가이드 (Installation & Execution)

### 서버 실행
프로젝트 폴더로 이동한 뒤, 아래 제공되는 스크립트를 통해 로컬 서버를 실행합니다.
스크립트 실행 시 필요한 필수 패키지(`Flask`, `Werkzeug`, `waitress`)가 없다면 자동으로 설치 여부를 묻고 진행합니다.

**Mac / Linux 환경**
최초 1회 실행 권한을 부여한 뒤, 쉘 스크립트 형태로 간편하게 실행할 수 있습니다.
```bash
chmod +x run.sh
./run.sh
```
*(기존 방식인 `python backend_app.py` 명령어로도 동일하게 실행 가능합니다.)*

**Windows 환경**
```bash
python backend_app.py
```

### 접속
웹 브라우저를 열고 다음 주소로 접속합니다.
```text
http://127.0.0.1:5000
```

## 5. 데이터 백업 및 보안 가이드 (Backup & Security)

### 데이터 백업 및 복구
*   **웹에서 간편하게 백업 (권장)**: 앱 화면 상단의 **[완벽 백업]** 버튼을 클릭하면 로그인한 계정의 전체 DB(`journal.db`)와 첨부 이미지가 포함된 무결성 검증 ZIP 파일이 다운로드됩니다. 새로운 환경에서 **[백업 원복]** 버튼을 통해 업로드하면 기존 상태 그대로 복원됩니다.
*   **자동 백업**: 매일 자정, 서버 내부(`backup/` 폴더)에 사용자별 백업 압축 파일이 자동 생성되며 100% 복원 가능한 상태인지 자체 무결성 검사(CRC, 레코드 수)를 수행합니다.
*   **수동 백업**: 직접 서버를 이전할 경우, 프로젝트 내의 `db/` 폴더와 `uploads/` 폴더만 복사하시면 됩니다.

### 보안 가이드 (Security Guide)
본 애플리케이션은 개인의 민감한 금융 및 투자 데이터를 다루므로 각별한 주의가 필요합니다.
*   **최고 관리자 계정**: 앱을 최초로 설치한 후 가장 처음으로 가입하는 계정이 자동으로 최고 관리자로 지정되며, 이후 가입하는 사용자는 이 관리자의 승인이 있어야 로그인이 가능합니다.
*   **세션 보안**: 내부적으로 세션 탈취(XSS) 및 크로스 사이트 요청 위조(CSRF) 방지 보안 쿠키 설정이 적용되어 있으며, 공용 기기에서의 데이터 유출을 막기 위해 1시간 동안 활동이 없으면 자동으로 로그아웃됩니다.
*   **외부 접속 주의**: 공유기 포트포워딩을 통한 직접적인 외부망(HTTP) 노출은 권장하지 않습니다. 외부(스마트폰 등)에서 안전하게 접속하려면 `ngrok`, `Cloudflare Tunnels`, `Tailscale`과 같은 암호화된 보안 터널링 서비스를 이용하시기 바랍니다.
*   **운영 환경(HTTPS)**: 로컬망에서는 `waitress`를 통해 안정적으로 구동되지만, 본격적으로 웹에 퍼블리싱 하시려면 Nginx 등의 리버스 프록시(Reverse Proxy)를 연동하여 HTTPS(SSL) 인증서를 적용하는 것을 적극 권장합니다.

## 6. 프로젝트 구조 (Project Structure)

```text
stock-memo/
├── backend_app.py      # 앱 구동·라우트·DB 헬퍼·백그라운드 스레드 (Flask)
├── prices.py           # 현재가(시세) 조회 서비스 (provider별 분해 + 폴백)
├── stats.py            # 매매 성과 분석(통계) 계산 로직 (순수 함수)
├── entry_logic.py      # 매매 기록 저장/무결성 검증 + INSERT 컬럼 단일 소스
├── backups.py          # 백업 ZIP 무결성 검증 로직 (순수 함수)
├── templates/          # 로그인·회원가입 HTML 템플릿
├── stock-memo.html     # 프론트엔드 메인 화면 구조 (HTML)
├── style.css           # 화면 디자인 및 레이아웃 (CSS)
├── calc.js             # 매매 계산 단일 소스 (백엔드 stats.py 와 동일 알고리즘)
├── script.js           # 화면 동작, 데이터 통신, 차트 로직 (JavaScript)
├── run.sh              # 자동화 실행 쉘 스크립트 (Mac/Linux)
├── backup/             # 매일 자동 생성되는 사용자별 백업 파일(ZIP) 저장 폴더
├── db/                 # 데이터베이스 폴더
│   └── journal.db      # 자동 생성되는 매매 기록 데이터베이스 파일 (SQLite)
├── logs/               # 시스템 및 에러 로그가 저장되는 폴더
│   └── backend_app.log # 디버깅/에러/경고 등 서버 실행 로그 파일
└── uploads/            # 첨부된 이미지 파일이 저장되는 폴더
```

> 백엔드는 도메인별 모듈(`prices`/`stats`/`entry_logic`/`backups`)로 분리되어 있으며,
> 손익 계산은 프론트엔드 `calc.js` 와 백엔드 `stats.py` 가 **동일한 이동평균단가
> 알고리즘**을 사용하도록 단일화되어 있습니다 (`tests/calc.test.js` 가 양쪽 일치를 검증).

## 7. 테스트 가이드 (Testing)

백엔드 API와 데이터 무결성 검증, 백업 복원(라운드트립), 성과 분석 로직은 `pytest` 기반 단위 테스트로 검증됩니다.
테스트 코드는 `tests/` 폴더에 위치하며, 임시 DB를 사용하므로 실제 구동 데이터(`db/journal.db`)에 영향을 주지 않습니다.

```bash
# 백엔드 전체 테스트 실행
pytest

# 간결한 출력 모드로 실행
pytest -q
```

프론트엔드 계산 엔진(`calc.js`)은 Node 내장 테스트 러너로 검증하며, 백엔드 통계
테스트와 동일한 픽스처를 사용해 프론트=백엔드 결과 일치를 보장합니다.

```bash
# 프론트엔드 계산 단위 테스트 (Node 18+)
node --test tests/calc.test.js
```
