# 📈 주식 매매 일지 (Stock Trading Journal) - Multi-User Edition

개인 투자자를 위한 **로컬 기반 주식 매매 일지 및 포트폴리오 관리 웹 애플리케이션**입니다.
파이썬(Flask) 백엔드와 SQLite를 기반으로 하며, **다중 사용자(Multi-User) 환경**과 **모바일 기기 최적화**가 적용되어 언제 어디서든 안전하고 쾌적하게 매매 기록을 관리할 수 있습니다.

## ✨ 주요 기능

- **🔐 다중 사용자 & 보안**: 회원가입 및 로그인 기능을 제공하며, 사용자별로 매매 기록과 첨부파일이 완벽하게 격리되어 안전하게 보관됩니다.
- **📝 매매 일지 & 메모 작성**: 매수, 매도, 관망 기록뿐만 아니라 종목에 대한 일반 메모와 아이디어를 기록할 수 있습니다.
- **📊 포트폴리오 대시보드**: 매매 기록을 바탕으로 현재 보유 종목, 총 투자 금액, 평균 단가, 누적 실현 손익을 자동 계산하고 파이 차트로 시각화합니다. (사용자 지정 드래그 앤 드롭 정렬 지원)
- **📅 캘린더 뷰**: 월별 달력에서 일자별 매수/매도/주시/메모 건수와 실현 손익을 색상 테마별로 한눈에 파악할 수 있습니다.
- **📰 실시간 뉴스 연동**: 현재 보유 중이거나 최근 기록한 종목과 관련된 실시간 뉴스를 구글 뉴스(RSS)에서 가져와 우측 사이드바에 5분 주기로 업데이트합니다.
- ** 시스템 로깅**: 서버 실행 시 발생하는 모든 이벤트(접속, 에러, 디버깅 정보 등)가 화면과 `logs/backend_app.log` 파일에 동시 기록되어 유지보수와 모니터링이 편리합니다.
- **🖼️ 스마트 에디터**: 에디터 본문 내에 이미지 직접 삽입(클립보드 스크린샷 붙여넣기, 드래그 앤 드롭 지원) 및 크기 조절이 가능하여 매매 차트나 증빙 자료를 손쉽게 기록할 수 있습니다.
- **📱 모바일 최적화 (PWA)**: iOS/Android 홈 화면 추가 시 네이티브 앱처럼 동작하며, '당겨서 새로고침(Pull-to-Refresh)' 및 터치 친화적인 UI를 제공합니다.
- **💾 개별 백업 및 복구**: 로그인한 사용자의 데이터(DB 및 업로드 파일)만 안전하게 ZIP 파일로 백업하고 복원할 수 있습니다.

## 🚀 설치 및 실행 방법

### 1. 필수 요구 사항
- Python 3.x 버전
- 웹 브라우저 (Chrome, Safari, Edge 등)
- (선택) 외부 접속을 위한 `ngrok` 및 `tmux`

### 2. 라이브러리 설치
터미널을 열고 아래 명령어를 입력하여 필요한 패키지를 설치합니다.
```bash
pip install Flask Werkzeug waitress
```

### 3. 서버 실행
프로젝트 폴더로 이동한 뒤, 아래 명령어를 통해 로컬 서버를 실행합니다.

**Mac / Linux 환경**
최초 1회 실행 권한을 부여한 뒤, 스크립트 형태로 간편하게 실행할 수 있습니다.
```bash
chmod +x backend_app.py
./backend_app.py
```
*(기존 방식인 `python app.py` 명령어로도 동일하게 실행 가능합니다.)*

**Windows 환경**
```bash
python backend_app.py
```

### 4. 접속
웹 브라우저를 열고 다음 주소로 접속합니다.
```text
http://127.0.0.1:5000
```

## 📁 프로젝트 구조

```text
stock-memo/
├── backend_app.py     # 백엔드 서버 구동 및 DB 관리 로직 (Flask)
├── stock-memo.html    # 프론트엔드 메인 화면 구조 (HTML)
├── style.css          # 화면 디자인 및 레이아웃 (CSS)
├── script.js          # 화면 동작, 데이터 통신, 차트 로직 (JavaScript)
├── db/                # 데이터베이스 폴더
│   └── journal.db     # 자동 생성되는 매매 기록 데이터베이스 파일 (SQLite)
├── logs/              # 시스템 및 에러 로그가 저장되는 폴더
│   └── backend_app.log # 디버깅/에러/경고 등 서버 실행 로그 파일
└── uploads/           # 첨부된 이미지 파일이 저장되는 폴더
```

## 🛠️ 데이터 백업 및 복구
- **웹에서 간편하게 백업 (권장)**: 화면 상단의 **[완벽 백업]** 버튼을 클릭하면 전체 DB(`journal.db`)와 첨부 이미지가 포함된 ZIP 파일이 다운로드됩니다. 새로운 환경에서 **[백업 원복]** 버튼을 통해 업로드하면 기존 상태 그대로 복원됩니다.
- **수동 백업**: 직접 서버를 이전할 경우, 프로젝트 내의 `db/` 폴더와 `uploads/` 폴더만 새로운 환경으로 복사하시면 모든 기록이 복구됩니다.

## 🛡️ 보안 가이드 (Security Guide)
본 애플리케이션은 개인의 민감한 금융 데이터를 다루므로 다음과 같은 보안 수칙을 권장합니다.
- **최고 관리자 계정 생성**: 앱을 최초로 설치한 후 가장 처음으로 가입하는 계정이 자동으로 최고 관리자로 지정되며, 이후 가입하는 사용자는 이 관리자의 승인이 있어야 로그인이 가능합니다.
- **세션 보안**: 애플리케이션 내부에 세션 탈취(XSS) 및 크로스 사이트 요청 위조(CSRF) 방지 보안 쿠키 설정이 적용되어 있으며, 공용 기기에서의 데이터 유출을 막기 위해 1시간 동안 활동이 없으면 자동으로 로그아웃됩니다.
- **외부 접속 주의**: 공유기 포트포워딩을 통한 직접적인 외부망(HTTP) 노출은 권장하지 않습니다. 외부(스마트폰 등)에서 안전하게 접속하려면 `ngrok`, `Cloudflare Tunnels`, `Tailscale`과 같은 암호화된 보안 터널링 서비스를 이용하시기 바랍니다.
- **운영 환경(HTTPS)**: 로컬망에서는 `waitress`를 통해 안정적으로 구동되지만, 본격적으로 웹에 퍼블리싱 하시려면 Nginx 등의 리버스 프록시(Reverse Proxy)를 연동하여 HTTPS(SSL) 인증서를 적용하는 것을 적극 권장합니다.
